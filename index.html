LLM Harness Design Patterns

Index — Ten Harnesses, Ten Writeups

A series of design-pattern write-ups, each extracting the generalizable LLM-harness tricks from a notable open-source project. Each row below points to the writeup, names the harness, summarizes why the harness is notable, and links to the canonical source so the claim can be verified. The cross-reference matrix across all ten lives at common.html.

Writeup Harness What it's famous for Sources
Part 1 tricks.html Poetiq ARC-AGI Solver poetiq-ai · 2025 Set the public state-of-the-art on ARC-AGI-1 and ARC-AGI-2 using off-the-shelf commercial LLMs (Gemini 3, GPT-5.1, Claude Haiku, Grok 4 Fast Reasoning) — a compact harness rather than a special model. Open-sourced the entire solver. Cited as evidence that harness design, not model capability alone, determines the leaderboard gap on ARC-AGI. repo launch post ARC Prize
Part 2 tricks2.html rStar-Math Microsoft Research · 2025 Showed that a 7B small language model, paired with MCTS-style deep thinking and a self-evolved process preference model, can match or exceed OpenAI o1 on competition math. The training apparatus is the headline contribution; the writeup pulls out the training-free harness tricks that transfer to any frozen API model. arXiv 2501.04519 MS Research community re-implementation
Part 3 tricks3.html LATS — Language Agent Tree Search Zhou et al. · ICML 2024 Unified reasoning, acting, and planning into one framework: MCTS where each node is an LLM rollout and the value function is the same frozen LLM behind a different prompt. Demonstrated across HotPotQA, HumanEval/MBPP, and WebShop with a single skeleton — the canonical reference for "tree search over LLM trajectories with verbal reflection memory." arXiv 2310.04406 project page repo
Part 4 tricks4.html FeatureBench evaluation harness · writeup pending on disk Evaluation harness for grading agent-produced patches against real test suites — the only non-inference harness in the series. Notable for its dual-patch test masking (the agent can't see the test surface; the grader sees it perfectly), per-repo test-parser dispatch, and two-mode grading (PASS_AND_FAIL vs FAIL_ONLY) for CI-flaky repos. Detailed cells in common.html; the standalone tricks4.html writeup is referenced by the series but not yet present on disk. common.html · FeatureBench column
Part 5 tricks5.html deepsearch-tts hkust-nlp Test-time-scaling research for web-grounded QA. Central finding: an asymmetric solver/verifier pair — same loop, same tools, different system prompts — benefits more from extra search budget on the verifier side than on the solver side. Also surfaces "budget forcing" (a user-role turn that flips the model from explore to commit) and condition-decomposition confidence scoring. repo
Part 6 tricks6.html DeepAgent openJiuwen-ai/agent-store · community Production-style long-horizon ReAct agent for tool-using QA. Notable for externalized plans (a regex-driven PlanTracker writes todo.md to disk and re-injects it next iteration so the plan survives context trimming), hierarchical agent budgeting with capability gating on sub-agents, and a staged final-answer extraction pipeline (hints → type → typed extraction) with per-stage retry budgets. repo
Part 7 tricks7.html AlphaCodium CodiumAI · 2024 Introduced "Flow Engineering" as an alternative to prompt engineering for code generation: a directed multi-stage pipeline (problem reflection → public-test reasoning → solution generation → ranking → AI-test generation → iterative fix) outperformed prior published results on CodeContests using off-the-shelf GPT-4 — without any fine-tuning. Established the now-common pattern of structured YAML handoffs between LLM-driven stages. arXiv 2401.08500 repo CodiumAI
Part 8 tricks8.html ARCgentica Symbolica AI ARC-AGI solver built on Symbolica's Agentica SDK. Demonstrates the opposite design philosophy from Poetiq's flat ensemble: an LLM-owned REPL loop with recursive sub-agent trees and clean-room re-verification. The harness is thin; the LLM is given a Python runtime with verifier helpers pre-imported and decides — in code — how wide and how deep to branch. Trades ensemble breadth for per-attempt agentic depth. Symbolica reported a state-of-the-art ARC-AGI-2 result in February 2026 using this approach. repo Symbolica blog Agentica SDK ARC Prize
Part 9 tricks9.html Reflexion Shinn et al. · NeurIPS 2023 Introduced "verbal reinforcement learning" — an LLM agent reflects on a failed attempt in natural language, and that paragraph is injected as memory on the next attempt. The foundational paper for self-reflective agent loops: every later framework that "writes a post-mortem after each trial and uses it as context" descends from this. Demonstrated across programming (HumanEval), decision-making (AlfWorld), and reasoning (HotPotQA). arXiv 2303.11366 repo author post
Part 10 tricks10.html Voyager NVIDIA / MineDojo · 2023 First lifelong-learning LLM agent in Minecraft. Introduced the canonical three-component recipe for open-ended LLM agents: automatic curriculum (let the agent propose its next sub-goal), iterative prompting with environment feedback (refine code on raw env errors and chat output), and an ever-growing skill library (verified executable code, embedded and retrieved for future tasks). Outperformed prior agents on novel-item discovery and exploration distance — the template most subsequent "agent with a skill library" works build on. arXiv 2305.16291 project page repo
